# AI-Assisted Image Analysis

Spot & Report can optionally use OpenAI to describe visible details in the
single photo selected for a bird report. The feature assists the reporter; it
does not diagnose disease, confirm species or replace human review.

## Observations

The structured response contains only:

- a possible bird type;
- an estimated visibly distinguishable bird count;
- an apparent condition;
- short visible concerns;
- a general environment description.

Every category has `high`, `medium` or `low` confidence. Species wording is
always provisional, and weak evidence produces uncertainty or `null`.

## Reporter approval

Analysis starts only when the reporter selects **Analyse photo**. Generated
observations are not part of the report until **Use these observations** is
selected. The reporter can discard or rerun the analysis, and replacing the
photo clears observations associated with the previous image.

The interface always states:

> These observations were generated by AI and may be incorrect. They are not a
> species confirmation, veterinary assessment, or disease diagnosis.

The report can continue and submit without image analysis.

## OpenAI boundary

The browser sends the selected photo to `POST /api/report/image-analysis`.
Only the server calls the OpenAI Responses API using `OPENAI_API_KEY` and an
optional `OPENAI_VISION_MODEL`. The default is `gpt-5.6-terra`.

OpenAI receives exactly:

- the selected JPEG, PNG or WebP image as one Base64 data URL;
- one short statement that it concerns a public sick or deceased bird report;
- safety instructions and the strict observation schema.

OpenAI does not receive GPS coordinates, location accuracy, manual location
text, reporter identity, reporter notes, report references, written summaries,
Supabase credentials or unrelated Report Session fields. Requests use
`store: false` and no tools or multi-turn context.

HEIC and HEIF remain valid report-photo formats but are not analysed because
OpenAI's current vision input formats do not include them. No image conversion
is performed in this MVP.

## Persistence

Only approved observations are submitted. Supabase stores the structured JSON,
model, generation timestamp and approval timestamp in nullable report columns.
Discarded and unapproved responses, prompts, raw provider responses and hidden
reasoning are not stored.

## Failure behaviour

Missing configuration, invalid images, unsupported formats, rate limits,
timeouts, malformed model output and provider failures return controlled
messages. Raw provider errors are not returned or logged. Failure never removes
the selected photo or blocks the reporter from continuing without AI.

## Known limitations

Visual observations can be wrong, especially with distant, obscured, rotated or
low-quality images. The feature does not identify disease, determine cause of
death, provide veterinary advice, analyse multiple photos or trigger agency
delivery.
